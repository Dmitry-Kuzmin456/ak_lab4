# Golden tests

Каждый файл `*.yml` содержит один интеграционный golden test для всей цепочки:

1. asm source транслируется в бинарный файл формата `AK4B`;
2. проверяется полный hex-образ бинарника, включая память команд и память данных;
3. создается debug dump с адресами, hex-кодом и мнемониками;
4. модель процессора запускается с входным потоком;
5. проверяется вывод, полный debug dump и trace.

Поля кейса:

- `in_source` -- исходная программа на asm.
- `in_input` -- входной поток.
- `in_limit` -- лимит тактов.
- `out.out_stdout` -- полный ожидаемый вывод.
- `out.out_preprocessed` -- ожидаемый результат препроцессинга, если кейс
  отдельно проверяет макросы и условную компиляцию.
- `out.out_binary_hex` -- полный ожидаемый бинарный образ `AK4B` в hex.
- `out.out_code_log` -- полный ожидаемый debug dump транслятора.
- `out.out_trace` -- ожидаемый журнал тактов; для `prob1` хранится только
  первые 1000 строк.

Для всех тестов, кроме `prob1`, trace сравнивается целиком. Для `prob1`
сравнивается префикс, потому что полный журнал слишком большой.
