# Golden tests

Каждый файл `*.yml` содержит один интеграционный golden test для всей цепочки:

1. asm source транслируется в бинарный файл формата `AK4B`;
2. создается debug dump с адресами, hex-кодом и мнемониками;
3. модель процессора запускается с входным потоком;
4. проверяется вывод, полный debug dump и ключевые строки trace.

Поля кейса:

- `in_source` -- исходная программа на asm.
- `in_input` -- входной поток.
- `in_limit` -- лимит тактов.
- `out.out_stdout` -- полный ожидаемый вывод.
- `out.out_code_log` -- полный ожидаемый debug dump транслятора.
- `out.out_trace_contains` -- обязательные строки журнала тактов.

Полный trace больших программ не хранится в эталоне. Вместо этого проверяются
устойчивые фрагменты: ввод-вывод, ключевые состояния регистров и отдельные такты.
