# Спецификация процессора

## Регистры

* R1-R3: 32-битные регистры данных общего назначения
* A1-A3: 16-битные адресные регистры
* ZERO: специальный регистр, чтение всегда возвращает 0, запись игнорируется
* IP: 16-битный instruction pointer
* CR: 8-битный регистр opcode
* PS: флаги N/Z/V/C

## Система команд

### Управление

* HLT
* NOP

### Работа с памятью и данными

* MOV src, dst

Операнды задают режим адресации:

* R1-R3 -- регистр данных
* A1-A3 -- адресный регистр
* label -- прямая память данных
* (A1), (A2), (A3) -- косвенная адресация через адресный регистр
* #value -- immediate
* ZERO -- специальный регистр

### Арифметика

* ADD src, dst
* SUB src, dst
* MUL src, dst
* DIV src, dst
* MOD src, dst
* POLY x, c0, c1, ..., cn, dst
* INC dst
* DEC dst
* CMP src, dst

### Команды перехода

* JMP addr
* BEQ addr
* BNE addr
* BLT addr
* BGT addr

### Ввод-вывод

* IN port, dst
* OUT src, port
* OUT_CSTR addr

## Ассемблер

* Метки
* Секции `.section text` и `.section data`
* Директива `.org`
* Константы `.const NAME VALUE`
* Данные `.word` и `.cstr`
* Макросы `.macro NAME [arg ...]` / `.endmacro`
* Условная компиляция `.ifdef`, `.ifndef`, `.ifconst`, `.else`, `.endif`

## Микропрограммное устройство

* Микрокод хранится в `src/microcode_memory.py`
* Микрокоманда задаёт `op`, `alu` и `move`
* Один tick выполняет одну микрокоманду

## Устройство памяти

### Память команд

* 2^16 = 65536 ячеек
* 8-битные ячейки
* обычная инструкция кодируется как opcode и список операндов
* каждый операнд кодируется как kind и 2-байтное значение
* `POLY` дополнительно хранит количество операндов и имеет переменную длину

### Память данных

* 2^16 = 65536 ячеек
* 32-битные ячейки
